# 角色
你是一位資深的全端工程師。請根據需求描述協助我設計一個網頁的「營業額提交及打卡系統」，目前第一版只需要提交營業額功能。

# 背景
我們是市場擺攤的商家，需要提交營業額，並且需要查看所有的營業額。
我們會在不同的市場擺攤，每個市場的擺攤位置不同，所以需要提交不同的地點。

# UI/UX 需求
- 前端框架：Vue3
- 前端語言：TypeScript
- 前端UI框架：PrimeVue
- 前端 CSS 框架：Tailwind CSS
- 風格： Y2K 風格。
- RWD：手機最為優先，因為大部分會使用手機提交營業額。

# 功能需求

## 驗證
- 需要輸入手機號碼進行驗證，驗證方式為：
    - 驗證方式：透過 Google Apps Script 傳送到 Google Sheets
    - 驗證邏輯：
        - 驗證是否為有效手機號碼

## 權限
- 有兩種權限：
    - 普通權限：
        - 只能提交營業額
        - 可以查看當日的營業額
    - 管理員權限：
        - 可以提交營業額
        - 可以查看所有的營業額


## 操作流程
- 普通權限使用者：
    - 首頁：輸入手機號碼進行驗證
    - 驗證成功後，可以提交營業額
    - 可以查看當日的營業額
- 管理員權限使用者：
    - 首頁：輸入手機號碼進行驗證
    - 驗證成功後，可以提交營業額
    - 可以查看所有的營業額

## Google Sheets 配置
使用者資料：
    - ID
    - 手機號碼
    - 權限

市場資料：
    - ID
    - 市場名稱
    - 市場位置
    - 市場租金

營業額資料：
    - ID
    - 營業額
    - 提交時間
    - 提交者
    - 地點
    - 備註

# 環境變數設定 (.env)
請將以下參數設計為可透過環境變數配置：
- GOOGLE_APP_SCRIPT_URL：Google Apps Script 的後端連結
